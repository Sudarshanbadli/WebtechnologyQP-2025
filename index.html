Q2
What is Hypertext link?
2 marks
▼

A Hypertext Link (also called a hyperlink) is a reference in a hypertext document that points to another document, section, image, or resource. When clicked, it navigates the user to the linked location.

In HTML, hyperlinks are created using the <a> (anchor) tag with the href attribute:

<a href="https://www.example.com">Click Here</a>
  • Links can point to external websites, internal pages, email addresses (mailto:), or file downloads.
  • They form the backbone of navigation on the World Wide Web.
  • target="_blank" opens the link in a new tab.

Q4
Give the syntax of XML structure.
2 marks
▼

XML (eXtensible Markup Language) is a self-descriptive markup language used to store and transport data. Its structure follows strict rules:

<?xml version="1.0" encoding="UTF-8"?> <root> <element attribute="value"> <child>Text Content</child> </element> </root>
  • Every XML document must have a single root element.
  • All elements must be properly closed and properly nested.
  • Tags are case-sensitive.
  • Attribute values must be in quotes.
  • The XML declaration (<?xml ... ?>) is optional but recommended.
Q5
Expand and define DTD.
2 marks
▼

DTD stands for Document Type Definition.

It defines the legal building blocks of an XML or HTML document — the structure and allowed elements, attributes, entities, and their data types.

Internal DTD

Declared inside the XML document itself using <!DOCTYPE root [ ... ]>

External DTD

Stored in a separate .dtd file and referenced via SYSTEM or PUBLIC keyword.

<!-- Internal DTD Example --> <!DOCTYPE note [ <!ELEMENT note (to, from, body)> <!ELEMENT to (#PCDATA)> <!ELEMENT from (#PCDATA)> <!ELEMENT body (#PCDATA)> ]>

Q7
List any four features of Javascript.
2 marks
▼
  • Interpreted Language: JavaScript is executed directly by the browser's JavaScript engine without prior compilation, making it fast to run and test.
  • Object-Oriented: Supports objects, prototypes, and classes (ES6+) allowing code reuse and encapsulation.
  • Event-Driven: Can respond to user actions (clicks, mouse moves, key presses) through event listeners, enabling interactive web pages.
  • Lightweight & Dynamic: Variables are loosely typed (dynamic typing) and the language is lightweight, making it easy to write and embed in HTML.
  • Platform Independent: Runs on any browser on any OS — write once, run anywhere.
  • Asynchronous Support: Supports async programming via callbacks, Promises, and async/await (AJAX, Fetch API).
Q8
What are general syntactic characteristics of Javascript?
2 marks
▼
  • Case Sensitive: myVar and MyVar are different identifiers.
  • Semicolons Optional: Statements can end with ; (ASI — Automatic Semicolon Insertion handles omissions).
  • Dynamic Typing: Variable types are determined at runtime. A variable can hold different types during execution.

Q10

onfocus

Fires when an element receives focus (e.g., when a user clicks on an input field or tabs into it). Used to highlight the active field.

Q13
Explain frames in HTML.
5 marks
▼

Frames in HTML allow a browser window to be divided into multiple sections (frames), each displaying a separate HTML document independently.

Key Tags
  • <frameset> — Replaces the <body> tag; defines how the window is split. Uses rows or cols attributes.
  • <frame> — Individual frame; uses src attribute to load an HTML file.
  • <noframes> — Content displayed in browsers that don't support frames.
  • <iframe> — Inline frame; embeds a frame inside normal HTML content (still widely used).
<frameset cols="30%,70%"> <frame src="nav.html" name="leftFrame"> <frame src="content.html" name="rightFrame"> </frameset> <!-- Inline Frame (modern approach) --> <iframe src="page.html" width="600" height="400"></iframe>
Frame Attributes
AttributeDescription
srcURL of the page to load in the frame
nameUsed as a target for links
scrollingyes / no / auto — scroll bar control
noresizePrevents user from resizing the frame
frameborder0 or 1 — show/hide frame border
Note The <frameset> element is deprecated in HTML5. <iframe> is the modern alternative for embedding content.

Q15
Discuss 'If-else' and 'Switch' statements in Javascript.
5 marks
▼

Conditional statements allow programs to make decisions and execute different code based on conditions.

If-Else Statement

Executes a block of code if a condition is true, otherwise executes the else block.

var marks = 75; if (marks >= 90) { console.log("Grade A"); } else if (marks >= 75) { console.log("Grade B"); // ← this runs } else if (marks >= 60) { console.log("Grade C"); } else { console.log("Fail"); }
Switch Statement

Evaluates an expression and matches it against multiple case values. More readable than long if-else chains when comparing one variable to many values.

var day = "Monday"; switch(day) { case "Monday": console.log("Start of work week"); break; case "Friday": console.log("Last working day"); break; case "Saturday": case "Sunday": console.log("Weekend!"); break; default: console.log("Weekday"); }
Important Always use break in switch cases. Without it, execution falls through to the next case. The default runs when no case matches.
Q16
Explain how events are handled from Button Elements.
5 marks
▼

Buttons are interactive elements that trigger JavaScript actions when interacted with. There are three main ways to handle button events:

Method 1: Inline Event Handler
<button onclick="alert('Clicked!')">Click Me</button> <!-- Calling a function --> <button onclick="submitForm()">Submit</button>
Method 2: DOM Property Assignment
<button id="myBtn">Click</button> <script> document.getElementById("myBtn").onclick = function() { document.getElementById("output").innerHTML = "Button Clicked!"; }; </script>
Method 3: addEventListener (Best Practice)
<button id="btn">Click Me</button> <p id="result"></p> <script> var btn = document.getElementById("btn"); var count = 0; btn.addEventListener("click", function() { count++; document.getElementById("result").innerHTML = "Clicked " + count + " times"; }); // Multiple events on same button btn.addEventListener("mouseover", function() { btn.style.background = "lightblue"; }); </script>
Why addEventListener? It allows multiple handlers on the same event, better separation of concerns (HTML and JS), and is easier to remove with removeEventListener().
Q17
With example explain how you move an element in Javascript.
5 marks
▼

Elements can be moved on a webpage using JavaScript by dynamically changing their CSS position properties (top, left) or using CSS transitions / setInterval for animation.

Method 1: Direct Style Manipulation
<div id="box" style="position:absolute; top:50px; left:50px; width:80px; height:80px; background:cyan;"></div> <button onclick="moveRight()">Move Right</button> <script> function moveRight() { var box = document.getElementById("box"); var current = parseInt(box.style.left) || 50; box.style.left = (current + 20) + "px"; } </script>
Method 2: Continuous Animation using setInterval
<script> var pos = 0; var box = document.getElementById("box"); var timer = setInterval(function() { pos += 2; box.style.left = pos + "px"; if (pos >= 300) { clearInterval(timer); // Stop when reached 300px } }, 10); // Every 10ms </script>
Method 3: CSS Transition + JS Class Toggle
.box { transition: left 1s ease; position: absolute; left: 0; } .moved { left: 300px; } // JS document.querySelector(".box").classList.add("moved");

Part B — Drag and Drop in JavaScript

HTML5 provides a native Drag and Drop API that allows elements to be dragged and dropped onto target elements using the draggable attribute and drag-related events.

Key Drag Events
EventFired OnDescription
dragstartDraggable elementWhen the user starts dragging
dragDraggable elementWhile element is being dragged
dragendDraggable elementWhen dragging stops
dragoverDrop targetWhen dragged element is over target (call preventDefault!)
dragenterDrop targetWhen dragged element enters target
dragleaveDrop targetWhen dragged element leaves target
dropDrop targetWhen element is dropped onto target
Complete Example
<!DOCTYPE html> <html> <head> <style> #dragBox { width: 100px; height: 100px; background: #00e5ff; cursor: grab; border-radius: 8px; display: flex; align-items: center; justify-content: center; font-weight: bold; } #dropZone { width: 250px; height: 250px; border: 3px dashed #888; margin-top: 20px; display: flex; align-items: center; justify-content: center; color: #888; } #dropZone.dragover { background: rgba(0,229,255,.1); border-color: #00e5ff; } </style> </head> <body> <div id="dragBox" draggable="true">Drag Me!</div> <div id="dropZone">Drop Here</div> <script> var dragItem = document.getElementById("dragBox"); var dropZone = document.getElementById("dropZone"); // Store the id of dragged element dragItem.addEventListener("dragstart", function(e) { e.dataTransfer.setData("text", e.target.id); }); // Must prevent default to allow drop dropZone.addEventListener("dragover", function(e) { e.preventDefault(); dropZone.classList.add("dragover"); }); dropZone.addEventListener("dragleave", function() { dropZone.classList.remove("dragover"); }); dropZone.addEventListener("drop", function(e) { e.preventDefault(); var id = e.dataTransfer.getData("text"); var elem = document.getElementById(id); dropZone.appendChild(elem); // Move element into drop zone dropZone.classList.remove("dragover"); dropZone.innerHTML = ""; dropZone.appendChild(elem); }); </script> </body> </html>
Key Point Always call event.preventDefault() in the dragover handler — by default, browsers don't allow dropping, so preventing default enables it. Use dataTransfer to pass data between drag source and drop target.
